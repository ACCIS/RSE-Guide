## Core principles

1. [Write code for readability, maintainability, and clarity](/styleguide/code_for_readability/)

2. [Write small functions that do _one thing_, and do that _one thing_ well](/styleguide/SRP/)

3. [Don't assume your code does what you think - test it.](/styleguide/testing/)

4. [Use IDEs and automatic tools](/styleguide/IDEs_and_tools/)

5. [Use Version Control](/styleguide/version_control/)
